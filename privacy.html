Privacy policy

Quota Peek stores monitor data locally and connects directly to provider billing endpoints.

Last updated: 2026-07-14

Overview

Quota Peek is a Windows application published by Amuin. It helps users monitor AI API balances, costs, and usage.

Data used by the app

The app stores user-provided API keys, monitor names, refresh preferences, and locally saved usage snapshots on the user's device.

API keys are stored with Windows protected local storage mechanisms and are not uploaded to Amuin-operated servers by the current app release.

Data not collected by the app

Quota Peek does not collect prompts, model responses, identity documents, phone numbers, photos, audio recordings, contacts, or precise location.

The app does not provide a cloud relay for AI usage queries in the current release.

Network behavior

When a user configures or refreshes a monitor, the app may connect directly from that Windows device to the billing or usage endpoints of the configured provider, such as OpenAI, DeepSeek, or Moonshot.

These network requests are initiated by user configuration or refresh actions.

Optional Codex integration

The Codex integration is off by default. When enabled, Quota Peek first queries the locally installed Codex CLI app-server for subscription quota information without receiving its OAuth token.

Users may separately enable an experimental local fallback. It reads only the access token and account ID from the local Codex auth.json file and sends a quota request directly to https://chatgpt.com/backend-api/wham/usage. It never reads or refreshes the refresh token and does not copy or modify the file.

Codex credentials are used temporarily in memory. They are not saved or logged, and they are not sent to Amuin or the Quota Peek developer. The local cache contains normalized quota values, reset times, optional credits, refresh time, data source, and sanitized status only.

Users can disable the integration or use “Disable and clear cache” in Experimental Features to remove local Codex quota data and withdraw fallback consent.

Optional Claude integration

The Claude integration is off by default and is used only to monitor subscription quota. When enabled, Quota Peek reads only the Claude Code OAuth access token from its local Windows Credential Manager entry or credentials file and sends a quota request directly to https://api.anthropic.com/api/oauth/usage. It does not read prompts, responses, conversations, source code, or project files.

If the OAuth quota request is unavailable, Quota Peek may run the locally installed Claude Code executable with /usage and parse only quota percentages and reset times. It does not persist the command transcript.

Claude credentials are used temporarily in memory. They are not saved or logged, and they are not sent to Amuin or the Quota Peek developer. The local cache contains only normalized quota values, reset times, refresh time, data source, and sanitized status.

Users can disable the integration or use “Disable and clear cache” in Experimental Features to remove local Claude quota data without changing Claude Code authentication data.

Optional AI Agent status Hooks

AI Agent status monitoring remains disabled until the user installs a Quota Peek Hook for the corresponding local Agent. Before installing, repairing, or uninstalling a Hook, Quota Peek shows the exact target path, lifecycle events, commands, backup behavior, local status path, stored fields, purpose, and privacy boundary, then waits for explicit confirmation. Cancelling the dialog does not write a file or create a backup.

The Hook targets are %USERPROFILE%\.codex\hooks.json for Codex and %USERPROFILE%\.claude\settings.json for Claude. Existing third-party Hook entries are preserved. Quota Peek adds, replaces, or removes only commands carrying its owned --source quotapeek-agent-status marker and creates a timestamped sibling backup before changing an existing settings file.

The Hook receiver stores only the Agent name, hashed task ID, state, event name, and timestamp in the app's local data directory. This information is used only to display running, waiting, completed, and idle states. An Agent without a fully installed Quota Peek Hook is not monitored or shown.

Quota Peek does not store prompts, replies, tool arguments, raw Hook payloads, credentials, authentication files, source code, or project contents, and it does not send Agent status records to Amuin or the Quota Peek developer.

Uninstalling a Hook removes only Quota Peek-owned commands. Existing local status records remain until the user selects “Reset status,” but they are not monitored or displayed after the Hook is removed.

Retention

Configuration data and stored snapshots remain on the local device until the user deletes them or removes the app.

API keys

API keys are stored on the user's device using Windows protected storage and are used only for the corresponding provider queries configured by the user.

Data sharing

Quota Peek does not sell personal data and does not include analytics, telemetry, cloud sync, or an Amuin-operated relay in the current release.

Deleting data

Users can delete accounts in the app to remove their local configuration, related snapshots, and saved secret references. Users can also disable Codex or Claude integrations and clear their managed monitors, cached quota snapshots, and local consent without changing Codex or Claude authentication data.

Uninstalling the app removes the installed application package. Remaining local app data can also be cleared from Windows app settings.
